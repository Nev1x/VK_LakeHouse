# Устав проекта LoftNavigator (ЛофтНавигатор)

> Создан из `docs/constitution.template.md` (seed LV_AGENT_TEAM) на разовой настройке 2026-07-19.
> Проект: локальная Data-платформа LakeHouse для данных о квартирах — приём файлов любых форматов
> (CSV/Excel/JSON/…), преобразование по слоям medallion, дашборды, выгрузка версионированных
> датасетов в MinIO/S3 для последующего обучения ML-модели (классификация «лофт / не лофт» по
> объявлениям и фото). Правки после принятия — только через процедуру поправок (Часть III).
> Сверка спек/PR — `docs/constitution-check.md` (читает Часть I этого файла).

Версия: 1.0.0 · Owner: владелец проекта (Aleksa Kogalenok)

---

## Часть I — Инварианты

#### I-1 — Data Sovereignty & Deployment Model (суверенитет данных)
**MUST.** Всё разворачивается on-prem на локальной машине владельца через Docker Compose.
Данные квартир (сырые файлы, таблицы Iceberg, датасеты) НЕ уходят во внешние SaaS, облачные
LLM или сторонние API. Внутренняя data-сеть (`data_net`: Trino, MinIO, PostgreSQL, loader)
наружу не публикуется; наружу смотрят только Grafana и явно решённые владельцем entrypoints.
- _Как проверить:_ спека/дифф не добавляет egress данных за пределы машины; compose не
  публикует порты data_net-сервисов без решения владельца.

#### I-2 — Data is Sacred (данные священны)
**MUST.** Слой raw/bronze immutable: исходные файлы в MinIO никогда не перезаписываются и не
удаляются; повторная загрузка того же файла идемпотентна (не плодит дубликаты). Невалидные
записи уходят в quarantine с причиной отбраковки — молча не дропаются. Iceberg snapshots не
expire и таблицы не переписываются без явного решения владельца. Append-only журналы
(память агентов, audit.md, журналы загрузок) не чистятся.
- _Как проверить:_ в диффе нет перезаписи/удаления raw-объектов, нет `DROP`/rewrite таблиц без
  бэкапа и решения владельца; у отбраковки есть quarantine-путь.

#### I-3 — Audit Immutability (неизменяемость аудита)
**MUST.** Каждый прогон ingestion/transform/export пишет append-only запись журнала
(источник, число строк принятых/отбракованных, версия схемы, итог). Журнал прогонов не
редактируется задним числом.
- _Как проверить:_ новая пайплайн-операция пишет запись журнала; в диффе нет UPDATE/DELETE
  по журнальным таблицам/файлам.

#### I-4 — Clean Architecture (чистая архитектура)
**MUST.** Данные текут строго по слоям medallion: `raw → bronze → silver → gold`; слой читает
только предыдущий слой. Доступ к таблицам — только через Trino/Iceberg-каталог: сервисы,
дашборды и экспорт НЕ читают parquet/metadata-файлы в MinIO напрямую (единственное исключение —
raw-объекты, которыми владеет ingestion loader). Код платформы: CLI → service → io-адаптеры
(trino/s3), бизнес-логика не импортирует драйверы напрямую.
- _Как проверить:_ дифф не читает данные в обход Trino/каталога и не пишет «через слой»
  (например, ingestion сразу в gold).

#### I-5 — Tenant/Access Isolation — НЕ ПРИМЕНЯЕТСЯ (осознанно, см. журнал)
Платформа single-user, работает локально у владельца; тенантов и внешних пользователей нет.
Слот сохранён под номером для совместимости ссылок; вернётся при появлении
многопользовательского доступа.

#### I-6 — Frozen Contracts (замороженные контракты)
**MUST.** Замороженные поверхности: схемы gold-витрин, формат и манифест ML-датасета экспорта,
схема журнала прогонов. Изменения — только additive либо новой версией (dataset v2, таблица v2)
по решению владельца; удаление/переименование колонок задним числом запрещено.
- _Как проверить:_ дифф не меняет замороженную схему in-place без versioned-решения владельца.

#### I-7 — Security by Default (безопасность по умолчанию)
**MUST.** Креды MinIO/PostgreSQL/Grafana — только env/`.env` вне git (compose читает из env);
секреты никогда не попадают в репо, логи, тикеты, скриншоты. SQL к Trino — только
параметризованный/экранированный (никакой конкатенации пользовательского ввода). Админ-консоли
(MinIO, Trino UI) не публикуются наружу без пароля. Корпоративного SSO нет (слот удалён —
см. журнал).
- _Как проверить:_ constitution-check по диффу; secret-скан хука зелёный; compose без
  plaintext-кредов.

#### I-8 — Graceful Degradation (деградация без падения)
**MUST.** Отказ опциональной подсистемы не роняет ядро: падение Grafana не блокирует
ingestion/transform; ошибка одного входного файла (или одной строки) не роняет батч — файл
помечается failed/quarantine, остальные обрабатываются; недоступность Trino даёт понятную
ошибку CLI, а не молчаливый частичный результат.
- _Как проверить:_ выключенная подсистема → остальное работает, ошибка видима оператору (I-9).

#### I-9 — Observability (наблюдаемость)
**MUST.** Structured-логи (JSON или key=value) у loader/transform/export; healthcheck у каждого
compose-сервиса; каждый прогон пайплайна имеет run_id, который сквозит в логах и журнале
прогонов (I-3). Метрики качества загрузок (принято/отбраковано) видны в Grafana; retention
логов ≥ 7 дней.
- _Как проверить:_ у новой фичи ошибочные пути логируются с run_id и видны в дашборде/журнале.

#### I-10 — Deploy Safety (безопасный деплой)
**MUST.** Разворачивание только через `docker compose up -d --build` (+ make-обёртки); изменения
схем таблиц — expand-then-deploy (сначала additive-миграция, потом код); перед разрушающей
операцией над каталогом/данными — бэкап; после деплоя — verify (healthchecks + smoke-запрос
через Trino); откат описан в спеке фичи.
- _Как проверить:_ дифф не вводит ручной шаг деплоя; у миграции схемы есть план отката.

#### I-11 — Production-Ready by Default (без MVP-заглушек)
**MUST.** Ноль stub/TODO/placeholder как финальное состояние; фича = код + тесты + документация;
workaround только помеченный `# TEMP: … см. #тикет`.
- _Как проверить:_ grep диффа на stub/TODO без тикета; DoD stage-4 зелёный.

#### I-12 — Root-Cause Discipline (дисциплина первопричины)
**MUST.** Любой баг — через systematic-debugging до доказанной первопричины; scope-the-class
(все оси повторения: другие форматы входа, другие слои, другие таблицы) + guard от регрессии.
Канон: `docs/bug-handling-process.md`.

#### I-13 — Zero-Trust Verification (никому не верь на слово)
**MUST.** Claim (агента, CI, репортёра, своей памяти) принимается только со свежим собственным
evidence, желательно на другой оси (например, «данные загружены» проверяется SELECT'ом через
Trino, а не логом загрузчика). «FIXED» без перепроверки не существует.

#### I-14 — Code Speaks, Not Comments (код, а не комментарии)
**MUST.** Самодокументируемый код; комментарий-обещание запрещён; комментарий допустим только
для ограничения, которое кодом не выразить.

#### I-15 — Data Integrity Under Concurrency (целостность под конкуренцией)
**MUST.** Записи в таблицы — только через ACID-транзакции Iceberg; конкурентные прогоны одного
пайплайна сериализуются (lock/lease на уровне loader'а); выборки для дашбордов/экспорта
bounded (лимиты/партиции, не бесконечный full-scan по мере роста данных).
- _Как проверить:_ два параллельных запуска одного пайплайна не портят таблицу и не дублируют
  данные.

#### I-16 — Requirements Traceability (прослеживаемость требований)
**MUST.** Каждое требование спеки имеет след в плане и проверку в приёмке (`plan-lint` +
requirements-matrix stage-4); NFR квантифицированы (`Product_agents/NFR_GUIDE.md`).

## Часть II — Доменные каноны

- Архитектура платформы и потоки данных: `docs/architecture.md` (создаётся фичей 001) +
  исходные наброски владельца (ВК.md, ДЗ.md — исторический контекст, приоритет у этого устава).
- Слои данных: raw (файлы как есть) → bronze (типизированный append) → silver
  (`apartments_clean` — единая схема квартир) → gold (витрины + features для ML).
- Backlog фич: `specs/001-lakehouse-infra` … `specs/006-ml-dataset-export` (intent-brief в
  каждом каталоге).
- Доменная ловушка №1: входные схемы заранее неизвестны — загрузчик обязан работать
  универсально (schema inference + маппинг-конфиг), а не хардкодить колонки одного источника.

## Часть III — Процедура поправок

Изменение инварианта: PR + bump версии (MAJOR — удаление/ослабление, MINOR — добавление/усиление,
PATCH — редактура) + запись в журнал ниже. Молчаливый обход запрещён (I-13). Precedence:
**устав > доменный канон > адаптер раннера > память агентов**.
