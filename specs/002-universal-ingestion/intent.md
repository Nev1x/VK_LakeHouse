# Intent-brief 002 — universal-ingestion

**Цель.** Универсальный ingestion-загрузчик (Python CLI): принимает файлы данных о квартирах в
разных форматах и грузит их в lakehouse (raw → bronze), не зная схему заранее.

**Scope.**
- CLI `loftnav ingest <файл|папка>`: форматы CSV, Excel (xlsx/xls), JSON/JSONL; архитектура
  ридеров расширяема (Parquet и др. — потом).
- Schema inference: входные схемы заранее неизвестны (доменная ловушка №1 устава) — типы
  выводятся, схема фиксируется в журнале прогона.
- Raw-копия исходного файла кладётся в MinIO как есть, immutable (устав I-2).
- Bronze: append в Iceberg-таблицы через Trino; эволюция схемы — additive.
- Валидация записей; невалидные — в quarantine с причиной, не дропаются молча.
- Идемпотентность: повторная загрузка того же файла (по контент-хэшу) не дублирует данные.
- Журнал прогонов (run_id, источник, принято/отбраковано, версия схемы) — append-only (I-3),
  structured-логи с run_id (I-9).
- Ошибка одного файла не роняет батч (I-8).

**Вне scope.** Нормализация домена (003), HTTP API загрузки (отложено решением владельца —
CLI на MVP).

**Зависимости.** 001.

**Готово, когда.** Демо-набор из CSV + Excel + JSON с разными схемами грузится одной командой;
данные видны SELECT'ом через Trino (zero-trust проверка, I-13); повторный запуск не меняет
счётчики строк; невалидные строки лежат в quarantine.
