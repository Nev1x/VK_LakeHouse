## Проход 1 → FAIL (retry 1)

Ядро подтверждено адверсарно: SQL-инъекции через имена колонок, значение ячейки и --source —
все отбиты (санитизация + bind-параметры); lock-конкурентность без гонки; идемпотентность ×3;
балансы демо сходятся. Найдено: **CRITICAL-1** — при сбое quarantine-вставки журнал писал
failed rows_ok=0, хотя bronze уже содержал закоммиченные строки, а отбракованная строка
терялась молча (нарушение NFR-003/I-2); **CRITICAL-2** — chunk-cap 4MB бессмыслен: клиент
инлайнит bind-параметры, query.max-length=1M символов, QUERY_TEXT_TOO_LARGE на обычных данных
(1000×1.2KB); WARN-1 pandas-мэнглинг заголовков в обход санитайзера; WARN-2 битый JSON при
обрезке raw_record; INFO×2.

## Fix-цикл (stage-3, kulibin)

Инкрементальные счётчики факта (finally пишет реально закоммиченное + error_message; rejects
чанково по ходу); оценка длины инлайнового SQL (бюджет 700K, max_field 200K → reject строки);
заголовки CSV читаются сами (col_N/_2 только через санитайзер); валидный JSON при обрезке
(_truncated/_original_bytes/_prefix, байтобезопасно); дубль col_N-логики убран. +5 тестов.
Checkpoint: fix-коммит, дерево чистое.

## Вердикт: **PASS** (retry 1 из 2)

Основание: матрица 27/27 DONE; CRITICAL 0 после фикс-цикла; тесты/линт/смок зелёные; журнал
честен под сбоем; инъекции отбиты адверсарно.

success 0/0 без таблицы; small-file compaction bronze (OPTIMIZE) — кандидат.
