# Intent-brief 003 — silver-normalization

**Цель.** Нормализация домена квартир: из разношёрстных bronze-таблиц — единая чистая
silver-таблица `apartments_clean`.

**Scope.**
- Единая silver-схема квартиры: площадь, цена, локация/район, метраж до метро, комнаты,
  этаж, ремонт/стиль, мебель, ссылки на фото, источник, метки времени.
- Маппинг-конфиг «входная схема → silver-поля» (декларативный, per-источник; новые источники
  добавляются конфигом, не кодом).
- Приведение типов и единиц (м², валюта, минуты до метро), нормализация справочных значений.
- Дедупликация объявлений (правило идентичности — определить в спеке).
- Механика: Trino SQL-файлы + Python-раннер (решение владельца), инкрементальная обработка
  только новых bronze-данных.
- Записи, не прошедшие нормализацию, — в quarantine с причиной (I-2).

**Вне scope.** Агрегаты/витрины (004), ML-фичи (004/006).

**Зависимости.** 001, 002.

**Готово, когда.** Демо-данные из трёх разных входных схем сходятся в одну `apartments_clean`
с валидными типами; повторный прогон идемпотентен; quarantine объясним.
