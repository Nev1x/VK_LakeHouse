# Stage 1 — Creative Team, отчёт (004-gold-marts)

Дата: 2026-07-20. Состав (делегированный): Creative Analyst + System Analyst.

## Решения
1. Материализованные Iceberg-таблицы (не VIEW) — скорость дашборда + аудит/frozen бесплатно.
2. Полный пересчёт (не инкрементальный) — агрегаты зависят от всей выборки, медиана
   инкрементально некорректна; demo-объём дёшев; трактовка «производный слой, воспроизводимый».
3. build-in-shadow + atomic RENAME swap (CREATE OR REPLACE — spike-кандидат) — дашборд не видит
   полу-пересчёт.
4. 3 витрины (district/segment/dynamics) + apartments_features — не больше (YAGNI).
5. Динамика — по _ingested_at silver (НЕ pipeline_runs — I-4).
6. content_hash gold-прогона = snapshot_id silver (воспроизводимая привязка, замыкает provenance
   к манифесту 006).
7. КЛЮЧЕВОЕ: метки лофт/не-лофт в данных НЕТ (grep пусто). is_loft — NULL-заготовка, эвристика
   через style ЗАПРЕЩЕНА (лже-таргет/утечка, I-11); features экспортирует признаки, разметка —
   вне платформы. Оба аналитика подняли независимо.
8. Documented-gap: высота потолков, материал стен, планировка, год постройки — физически нет в
   silver, не выдумывается (I-6/I-11); добавление — additive-версией silver→gold.
