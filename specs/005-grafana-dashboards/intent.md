# Intent-brief 005 — grafana-dashboards

**Цель.** Дашборды Grafana поверх Trino: наблюдаемость платформы и аналитика по квартирам.

**Scope.**
- Trino datasource в Grafana (provisioning-конфигом, не руками; креды через env — I-7).
- Дашборд «Операции платформы»: прогоны ingestion (run_id, принято/отбраковано, quarantine),
  свежесть данных по слоям, ошибки (I-9).
- Дашборд «Квартиры»: агрегаты из gold-витрин (цены/площади/районы/стили).
- Дашборды — как код (provisioning JSON в репо), генерённые файлы руками не правятся.
- Падение/отсутствие Grafana не влияет на пайплайны данных (I-8).

**Вне scope.** Алёртинг за пределами базового (можно advisory), пользовательские роли.

**Зависимости.** 001 (datasource), 004 (витрины); операционный дашборд частично возможен уже
после 002.

**Готово, когда.** Оба дашборда поднимаются из provisioning с нуля (`make up`), показывают
живые данные; browser-QA (renata/semiglazka) проходит по ним без блокеров.
