# Intent-brief 006 — ml-dataset-export

**Цель.** Версионированный экспорт датасета для обучения ML-модели (классификация
«лофт / не лофт» по объявлениям и фото) в отдельный bucket MinIO/S3.

**Scope.**
- CLI `loftnav export-dataset`: собирает датасет из gold (`apartments_features`) —
  parquet и/или jsonl + ссылки/копии фото.
- Версионирование: каждый экспорт — новая версия `datasets/vNNN/` с манифестом
  (дата, run_id, число записей, схема, snapshot Iceberg-источника); старые версии
  не перезаписываются (I-2), формат манифеста — замороженный контракт (I-6).
- Отдельный bucket (например `ml-datasets`), не смешивается с warehouse.
- Данные не покидают машину владельца (I-1): «S3» = локальный MinIO; внешний S3 — только
  явным решением владельца отдельной фичей.

**Вне scope.** Само обучение/разметка модели; сервинг модели.

**Зависимости.** 004.

**Готово, когда.** Экспорт воспроизводим (одинаковый вход → одинаковый датасет), манифест
полон, версия читается независимым инструментом (pandas/pyarrow) без обращения к платформе.
